Este post es posiblemente el último post en este blog, ya que estoy en proceso de migración de plataforma. Razón de ello es tener una mayor libertad en las herramientas, posibilidad de automatizar mucho más simple, escritura sencilla y soporte simple para LaTeX y coloreado de sintaxis.

Igualmente la posibilidad de contar con control de versiones y sencilla gestión de dominio propio.

Seguramente este no será el mejor manual de migración, ya que ha sido un proceso lento por la falta de tiempo leyendo y explorando opciones, y considerando que por la misma falta de tiempo hay ciertas curvas de aprendizaje que solo por capricho evitaría.

Como restricciones aparecen un uso extendido de python como lenguaje principal en los últimos años, por lo que descarto plataformas en otros lenguajes (o si son en otro, con un buen binding para python). Esto entre otras descarta la popular Jekyll, soportada oficialmente por github basada en Ruby.

Nikola, Pelican, Tinkerer y MkDocs me parecieron alternativas interesantes entre las muchas disponibles en python, y esta una herramienta un paso más abajo para un mayor control que es pandoc (es en haskell, pero existen bindings en python).

Las entradas de mis dos blogs no las quiero perder, no simplemente no borrar los blogs sino portar las entradas, así que eso plantea una primera etapa.

En esto, solo Nikola ofrece opción de importar de blogger pero lo hace a partir del archivo xml de backup de blogger y genera los archivos listos para publicar (es bueno) pero no genera los archivos en markdown (o la fuente de preferencia), solo los archivos de metadatos.

Esto lleva a pensar en otra opción de mayor control, justo como pandoc. Pandoc es un conversor de documentos, y entre otras muchas opciones permite pasar de html a markdown, pero solo apunta a una pagina especifica. Se hace necesario elaborar un script que recorra todo el blog.
